30 ## Installation ##
31 
32 
33 The code should be compiled with gfortran 4.7+ (allows for allocatable arrays
34 in namelists). Unpack the archive in a folder, and run:
35  make
36 
37  Remark: The command "make clean" removes the compiled files.
38 
39 For Windows users, a minimalistic GNU development environment
40  (including gfortran and make) is available at [www.mingw.org](http://www.mingw.org) .

44 ## Description of the files ##
45 
46 The model tendencies are represented through a tensor called aotensor which
47 includes all the coefficients. This tensor is computed once at the program
48 initialization.
49 
50 * maooam.f90 : Main program.
51 * aotensor_def.f90 : Tensor aotensor computation module.
52 * IC_def.f90 : A module which loads the user specified initial condition.
53 * inprod_analytic.f90 : Inner products computation module.
54 * integrator.f90 : A module which contains the integrator for the model equations.
55 * Makefile : The Makefile.
56 * params.f90 : The model parameters module.
57 * maooam_tl_ad.f90 : Tangent Linear (TL) and Adjoint (AD) model tensors definition module
58 * tl_ad_integrator.f90 : Tangent Linear (TL) and Adjoint (AD) model integrators module
59 * test_tl_ad.f90 : Tests for the Tangent Linear (TL) and Adjoint (AD) model versions
60 * README.md : The present file.
61 * LICENSE.txt : The license text of the program.
62 * util.f90 : A module with various useful functions.
63 * stat.f90 : A module for statistic accumulation.
64 * params.nml : A namelist to specify the model parameters.
65 * int_params.nml : A namelist to specify the integration parameters.
66 * modeselection.nml : A namelist to specify which spectral decomposition will be used.
67 
68 A documentation is available in the folder `./doc'.

72 ## Usage ##
73 
74 The user first has to fill the params.nml and int_params.nml namelist files according to their needs.
75 
76 The modeselection.nml namelist can then be filled :
77 * params::nboc and params::nbatm specify the number of blocks that will be used in respectively the ocean and
78  the atmosphere. Each block corresponds to a given x and y wavenumber.
79 * The params::oms and params::ams arrays are integer arrays which specify which wavenumbers of
80  the spectral decomposition will be used in respectively the ocean and the
81  atmosphere. Their shapes are OMS(NBOC,2) and AMS(NBATM,2).
82 * The first dimension specifies the number attributed by the user to the block and the second
83  dimension specifies the x and the y wavenumbers.
84 * The VDDG model, described in Vannitsem et al. (2015) is given as an example
85  in the archive.
86 * Note that the variables of the model are numbered according to the chosen
87  order of the blocks.
88 
89 
90 Model and integration parameters can be specified in the params.nml namelist file.
91 
92 Finally, the IC.nml file specifying the initial condition should be defined. To
93 obtain an example of this configuration file corresponding to the model you
94 have previously defined, simply delete the current IC.nml file (if it exists)
95 and run the program :
96 
97  ./maooam
98 
99 It will generate a new one and start with the 0 initial condition. If you want another
100 initial condition, stop the program, fill the newly generated file and restart :
101 
102  ./maooam
103 
104 It will generate two files :
105  * evol_field.dat : the recorded time evolution of the variables.
106  * mean_field.dat : the mean field (the climatology)
107 
108 The tangent linear and adjoint models of MAOOAM are provided in the
109 maooam_tl_ad and tl_ad_integrator module. It is documented [here](./md_README_TL_AD.html).

114 ## Implementation notes ##
115 
116 As the system of differential equations is at most bilinear in y[j] (j=1..n), y
117 being the array of variables, it can be expressed as a tensor contraction
118 (written using Einstein convention, i.e. indices that occur twice on one side
119 of an equation are summed over):
120 
121  dy / dt = T y y (y == 1)
122  i i,j,k j k 0
123 
124 
125 The tensor aotensor_def::aotensor is the tensor T that encodes the differential equations
126 is composed so that:
127 
128 * T[i][j][k] contains the contribution of dy[i]/dt proportional to y[j]*y[k].
129 * Furthermore, y[0] is always equal to 1, so that T[i][0][0] is the constant
130 contribution to var dy[i]/dt.
131 * T[i][j][0] + T[i][0][j] is the contribution to dy[i]/dt which is linear in
132 y[j].
133 
134 Ideally, the tensor aotensor_def::aotensor is composed as an upper triangular matrix
135 (in the last two coordinates).
136 
137 The tensor for this model is composed in aotensor_def and uses the
138 inner products defined in inprod_analytic .
